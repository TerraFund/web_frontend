# Security Policy

## Supported Versions

We actively support the following versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in TerraFund, please report it to us as follows:

1. **Do not** create a public GitHub issue
2. Email security@terrafund.com with details
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Process

1. We will acknowledge receipt within 24 hours
2. We will investigate and provide an update within 7 days
3. We will keep you informed of our progress
4. Once fixed, we will publicly disclose the vulnerability

## Security Measures

TerraFund implements several security measures:

- **Authentication**: JWT-based authentication with secure token handling
- **Authorization**: Role-based access control (RBAC)
- **Data Protection**: Encryption of sensitive data at rest and in transit
- **Input Validation**: Comprehensive input sanitization and validation
- **Payment Security**: PCI-compliant payment processing
- **KYC Verification**: Identity verification for all users
- **Audit Logging**: Comprehensive logging of security events

## Responsible Disclosure

We kindly ask that you:

- Give us reasonable time to fix the issue before public disclosure
- Avoid accessing or modifying user data
- Do not perform DoS attacks or degrade service performance
- Do not spam our systems
